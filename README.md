## 介绍

Bangumi Obsidian Plugin 将 [bgm.tv](https://bgm.tv) 的条目数据（动画、书籍、游戏、音乐、三次元）导入 Obsidian，自动生成结构化笔记。

**主要能力：**

- 在线搜索（调用 Bangumi v0 API）或离线搜索（本地 JSONL 数据包）
- 自动填充封面、评分、infobox、制作人员、声优、分集/曲目列表
- 支持自定义 Mustache 风格模板（`{{变量}}`）
- 笔记更新时保留用户已有的手写内容（观看记录、个人总结等）

## 安装指南

### 用户安装（手动）

1. 前往 [GitHub Releases](https://github.com/kiwimichelle/obsidian-bangumi-plugin/releases/latest) 下载最新版本的 `main.js`、`manifest.json`、`styles.css`。
2. 在你的 Obsidian Vault 目录下找到 `.obsidian/plugins/`，新建文件夹 `bangumi-obsidian`。
3. 将三个文件放入该文件夹。
4. 打开 Obsidian → **设置 → 第三方插件** → 关闭「安全模式」→ 刷新列表 → 启用 **Bangumi**。

> **注意：** Obsidian 的「安全模式」必须关闭才能加载社区插件。如果你在公司设备或有安全顾虑的环境使用，请先了解相关风险。

---

### 开发者安装

#### 环境要求

- Node.js 20.x 或 22.x
- npm

#### 步骤

```bash
# 1. 克隆仓库
git clone https://github.com/kiwimichelle/obsidian-bangumi-plugin.git
cd obsidian-bangumi-plugin

# 2. 安装依赖
npm install

# 3. 开发模式（监听文件变更，自动编译到本地 Obsidian 插件目录）
npm run dev

# 4. 生产构建（输出 main.js 到项目根目录）
npm run build

# 5. 代码检查
npm run lint
```

> **开发模式与生产构建的区别：**
>
> - `npm run dev`：输出带 sourcemap 的调试版本，直接写入 `esbuild.config.mjs` 里配置的本地 Obsidian 插件目录（`targetDir`），**不生成** `main.js`。
> - `npm run build`：输出压缩后的生产版本到项目根目录的 `main.js`，用于发布 Release。
>
> **在发布前务必执行 `npm run build`，不要将开发版本（含 sourcemap）上传至 Release。**

#### 配置本地开发目录

打开 `esbuild.config.mjs`，修改 `targetDir` 为你本地 Obsidian Vault 中对应的插件路径：

```js
// esbuild.config.mjs
const targetDir = "D:/Obsidian/YourVault/.obsidian/plugins/bangumi-obsidian";
```

> **Windows 路径提示：** 反斜杠 `\` 在 JS 字符串里需要转义，建议统一改用正斜杠 `/`。

## 核心功能使用说明

### 搜索并建档

1. 点击左侧功能区的 📺 图标，或使用命令面板（`Ctrl/Cmd + P`）执行「搜索 Bangumi 条目」。
2. 在弹窗顶部选择搜索模式：**离线模式** 或 **在线模式**。
3. 点击类型筛选按钮（全部 / 动画 / 书籍 / 游戏 / 音乐 / 三次元）缩小范围。
4. 输入关键词，按 `Enter` 搜索。
5. 点击目标条目，填写主观信息（观看状态、评分、进度等），点击「保存并建档」。
6. 插件自动创建笔记，并在新标签页中打开。

**在线模式**会同时获取：条目基础数据、关联条目（续集/前传）、声优（角色+CV）、分集/曲目列表（四路并行请求，减少等待时间）。

---

### 离线模式

离线模式需要先下载 [Bangumi Archive](https://github.com/bangumi/Archive/releases/latest) 的数据包，配置路径后构建本地索引。

#### 数据包说明

| 文件名 | 大小（约） | 作用 | 是否必须 |
|--------|-----------|------|---------|
| `subject.jsonlines` | 300MB+ | 主条目数据，提供基础搜索和建档 | **必须** |
| `episodes.jsonlines` | 500MB+ | 分集数据，启用带集名/日期的 checkbox | 可选 |
| `persons.jsonlines` | 100MB+ | 人物数据，与条目人员配套提供离线制作人员 | 可选（须配套） |
| `subject-persons.jsonlines` | 200MB+ | 条目↔人员关联，须与 `persons` 一起配置 | 可选（须配套） |
| `subject-relations.jsonlines` | 50MB+ | 条目间关联（续集/前传/系列） | 可选 |

> **`persons` 与 `subject-persons` 必须同时配置才有效**，只配置其中一个不会生成制作人员数据。

#### 构建索引

1. 进入 **设置 → Bangumi → 离线数据包配置**，填写各数据包的本地绝对路径（或点击 📂 按钮浏览文件）。
2. 点击「🔄 立即构建」按钮。
3. 构建完成前可点击「在后台继续」关闭进度弹窗，插件在后台继续构建。

> **首次构建主条目索引（300MB+ 的 JSONL）约需 2–5 分钟**，取决于磁盘性能。构建期间 Obsidian 可正常使用，不会卡顿（分批让出主线程）。

> **替换数据包后必须重建索引**，否则行号索引失效，搜索结果可能指向错误的条目。

---

### 在线模式

在线模式不需要任何本地数据包，直接调用 bgm.tv API。

- 无需配置即可使用，但搜索速度受网络影响。
- 配置 API Token 可突破频率限制并访问私密数据（Token 在 bgm.tv 设置页生成）。
- 如未配置离线数据包，插件启动时会引导你选择模式。

---

### 更新已有笔记

对已建档的条目再次搜索并选择，插件会识别同一 `bangumi_id`，进入更新流程：

- **自动覆盖的内容：** Bangumi 元数据（评分、排名、简介、infobox、标签、封面）
- **自动保留的内容：**

| 分类 | 保留字段 |
|------|---------|
| 动画 / 三次元 | 已观看集数、观看网址、🎞️ 分集随笔（旧记录追加到新 checkbox 后）、个人总结 |
| 书籍 | 阅读状态、阅读进度、📝 读书随笔（追加）、个人总结 |
| 游戏 | 游玩状态、游玩时长、游玩平台、当前进度、📝 游玩随笔（追加）、个人总结 |
| 音乐 | 收听状态、收听平台、收听笔记（追加）、个人总结 |

> **分集随笔采用追加模式：** 新生成的 checkbox 列表会出现在前面，你之前写的旧笔记追加在后面，已勾选的记录不会丢失。

---

### 文件名冲突处理

当目标目录已存在同名文件时，根据**同名冲突策略**设置处理：

- **每次询问（默认）：** 弹出冲突解决弹窗，显示冲突文件路径和建议文件名（可编辑），你可以选择「覆盖已有文件」或「新建为此名称」。输入框实时验证文件名可用性。
- **自动覆盖更新：** 直接覆盖，不弹窗。
- **跳过不处理：** 直接跳过，不创建也不修改。

> **跨媒介冲突：** 如果同名文件在其他分类目录下（例如同名的书籍和动画），弹窗同样会出现，建议在文件名中加入类型标识（如「xxx (动画)」）。

## 配置项解析

进入 **设置 → Bangumi** 查看所有配置。

### 核心配置

| 配置项 | 说明 |
|--------|------|
| **Bangumi API Token** | 可选。填写后解锁更高频率限制和私密数据访问 |
| **优先使用离线模式** | 开启后搜索优先走本地索引；关闭则直接请求 API。也可在搜索弹窗内每次手动切换 |

---

### 离线数据包配置

参见[数据包说明](#数据包说明)和[构建索引](#构建索引)章节。

设置页的**索引状态面板**会显示每个索引的就绪状态：

- `✅ 已就绪`：索引已构建，可正常使用
- `⚠️ 未构建`：路径已配置但索引未生成，需点击「立即构建」
- `—（未配置路径）`：该数据包未配置，对应功能不可用

---

### 分类归档与模板

每种分类（动画/书籍/游戏/音乐/三次元）独立配置。

| 配置项 | 说明 |
|--------|------|
| **归档根目录** | 笔记保存的 Vault 内相对路径，如 `Bangumi/Anime` |
| **归档层级策略** | 仅动画/三次元可用。`年份/季度新番`：按 `2024/01月` 分目录；`仅年份`：按 `2024` 分目录；`无子目录`：直接放根目录 |
| **模板来源** | `默认模板`：使用内置模板；`自定义模板文件`：读取 Vault 内的 Markdown 文件作为模板 |
| **自定义模板文件路径** | 模板来源为「自定义」时生效，填写 Vault 内相对路径，如 `Templates/anime.md` |
| **同名冲突策略** | 见[文件名冲突处理](#文件名冲突处理) |

点击「👁️ 预览当前模板渲染效果」可查看模板渲染后的示例输出。

## 模板变量参考

在自定义模板中使用 `{{变量名}}` 语法，值为空时该变量会被替换为空字符串，同时模板引擎会自动清除空值导致的空表格行。

### 基础信息

| 变量 | 说明 |
|------|------|
| `{{title}}` | 条目中文名 |
| `{{original_title}}` | 条目原文名（日文名等） |
| `{{cover_local}}` | 封面本地路径，未下载时为远程 URL |
| `{{bangumi_id}}` | Bangumi 条目 ID |
| `{{bangumi_url}}` | Bangumi 页面链接 |
| `{{score}}` | 评分（0 时留空） |
| `{{rank}}` | 排名（0 时留空） |
| `{{summary}}` | 简介正文 |
| `{{today}}` | 建档日期（`YYYY-MM-DD`） |
| `{{year}}` | 开播/发售年份 |
| `{{season}}` | 开播季度（`01月` / `04月` / `07月` / `10月`） |
| `{{eps_count}}` | 集数（0 时留空） |

### 标签与 infobox

| 变量 | 说明 |
|------|------|
| `{{tags_yaml}}` | YAML 格式标签列表（`bangumi` + `bgm/标签名`） |
| `{{infobox_table_rows}}` | infobox 数据表格行（已自动过滤人员类字段） |
| `{{infobox_frontmatter}}` | infobox 数据的 frontmatter 格式 |

### 制作人员与声优

| 变量 | 说明 |
|------|------|
| `{{credits_main}}` | 主创人员表格行（导演/脚本/作画监督等） |
| `{{credits_cast}}` | 声优表格行，在线模式格式为 `角色 \| 声优`，离线模式仅显示声优名 |
| `{{credits_frontmatter}}` | 制作人员 frontmatter 格式 |

> **声优数据说明：** 在线模式下会请求 `/v0/subjects/:id/characters` 端点，获取角色名+CV 名对应关系。离线模式下若构建了 `subject-persons` 索引，仅显示声优名，无角色名。

### 分集与关联

| 变量 | 说明 |
|------|------|
| `{{eps_checkboxes}}` | 分集/曲目进度 checkbox 列表。在线模式含集名和播出日期；离线模式若有分集索引同样含集名；无数据时退化为纯序号 |
| `{{netaba_iframe}}` | Netaba.re 嵌入 iframe（动画专用） |
| `{{series_section}}` | 系列关联章节，有续集/前传/所属系列时自动生成 `## 系列关联` 表格，否则为空 |

### 分类特有

| 变量 | 适用分类 | 说明 |
|------|---------|------|
| `{{adaptation}}` | 动画 | 改编来源类型（原创/漫画改编/小说改编等） |
| `{{artist}}` | 音乐 | 艺术家名称 |
| `{{track_count}}` | 音乐 | 曲目数 |

### 主观输入

| 变量 | 适用分类 |
|------|---------|
| `{{my_status}}` | 全部 |
| `{{my_rating}}` | 全部 |
| `{{my_comment}}` | 全部 |
| `{{my_progress}}` | 动画、三次元 |
| `{{my_source}}` | 动画、三次元 |
| `{{my_read_progress}}` | 书籍 |
| `{{my_channel}}` | 书籍 |
| `{{my_version}}` | 书籍 |
| `{{my_hours}}` | 游戏 |
| `{{my_platform}}` | 游戏 |
| `{{my_game_progress}}` | 游戏 |
| `{{my_music_source}}` | 音乐 |

## 常见问题 FAQ

### 搜索没有结果

**离线模式：**
- 检查设置页主索引状态是否为 `✅ 已就绪`，未就绪则需要重建索引。
- 尝试更换关键词——离线搜索使用 CJK bigram 分词，搜索「魔法少女」比搜索「魔」命中率更高。
- 如搜索一个单字无结果，这是正常现象，单字无法构成 bigram，建议至少输入两个字。

**在线模式：**
- 检查网络是否能访问 `api.bgm.tv`。
- 配置 API Token 可减少因频率限制导致的搜索失败。

---

### 封面图片没有下载到本地

在线模式下，封面从 `lain.bgm.tv` 下载。可能的原因：
- 网络无法访问 `lain.bgm.tv`。
- 封面保存路径（`coverPath`）对应的目录没有写权限。

封面下载失败时，插件会退级使用远程 URL，笔记仍可正常创建，封面以网络图片形式展示。

---

### 建档后笔记里声优一栏是空的

| 情况 | 原因 | 解决方案 |
|------|------|---------|
| 在线模式，声优空白 | 该条目在 bgm.tv 没有角色数据，或角色数据无 CV | 属正常现象 |
| 离线模式，声优空白 | 未配置 `persons` + `subject-persons` 索引，或未重建 | 配置路径后重建索引 |

---

### 分集 checkbox 只有序号，没有集名

- **在线模式：** 分集数据请求失败（网络问题），退化为纯序号。
- **离线模式：** 未配置 `episodes.jsonlines` 路径或未重建分集索引。配置路径后重建索引即可。

---

### 更新笔记后分集随笔内容丢失

本插件采用**追加模式**保留分集随笔：新 checkbox 列表在前，旧内容追加在后。如果旧内容消失，可能是：
- 上次建档时使用的是旧版插件（保留策略不同），第一次用新版更新时旧内容无法被识别。
- 笔记的分集随笔标题被手动修改（必须与模板中的标题完全一致，如 `# 🎞️ 分集随笔`）。

---

### 替换了数据包文件，但搜索还是旧数据

索引没有自动失效检测（仅检测文件大小变化）。替换数据包后，**必须手动点击设置页的「🔄 立即构建」重建索引**。

---

### GitHub Actions 发布 Release 失败，报 403

在 `.github/workflows/release.yml` 的 job 下添加权限声明：

```yaml
permissions:
  contents: write
```

详见[安装指南 · 开发者安装](#开发者安装)章节。

---

### ESLint 报错：cannot find plugin `@typescript-eslint`

这是 ESLint 9 flat config 的已知问题：rules 对象里用到的插件必须在**同一个对象**内声明。确保 `eslint.config.js` 中的规则覆盖块包含：

```js
{
  plugins: {
    '@typescript-eslint': tseslint.plugin,
  },
  rules: {
    '@typescript-eslint/no-unused-vars': 'warn',
    // ...
  }
}
```

---

### 插件加载时 Obsidian 卡顿

首次加载时会并行读取全部索引文件（可能达数十 MB 的 JSON）。如果你配置了多个数据包索引，加载时间会相应增加。这是一次性开销，加载完成后搜索响应极快。如需优化启动速度，可暂时不配置不常用的可选索引（如 `persons`/`subject-persons`）。
